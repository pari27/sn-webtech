Authentication

All API requests require an API Key for authentication. The API Key uniquely identifies your account and authorizes your requests. You can find your API Key in the SMS Gateway dashboard under your user profile.

🔑
API Key Format: Your API key starts with sk_ followed by a 48-character hex string. Example: sk_a1b2c3d4e5f6...
How to send your API Key

You can pass the API Key in any of these three ways:

Method How to Send Example
Query Parameter Append ?api_key=YOUR_KEY to the URL /api/send.php?api_key=sk_xxx
POST Body Include api_key as a form field api_key=sk_xxx
HTTP Header Send as X-Api-Key header X-Api-Key: sk_xxx
⚠️
Security Warning: Never expose your API Key in client-side code (browser JavaScript). Always make API calls from your backend server.

Base URL

All API endpoints are relative to your gateway's base URL:

Base URL 
https://yourdomain.com/sms-gateway

Replace yourdomain.com/sms-gateway with the actual domain where your SMS Gateway is hosted. All API calls shown in this document use {BASE_URL} as a placeholder for this.

Send SMS

This is the primary endpoint to send a single SMS message to a mobile number.

POST GET {BASE_URL}/api/send.php
Sends a single SMS. Both GET (query params) and POST (form body) methods are supported. POST is recommended for security.

SMS Types (DLT)

India's TRAI mandates DLT (Distributed Ledger Technology) compliance. Choose the correct sms_type for your message:

sms_type ValueUse CaseTime Restriction (IST)
transactional ⭐ OTP, transaction alerts, login codes, password reset — sent to opted-in users ✅ 24×7 allowed
service_implicit Service messages to existing customers (bills, delivery updates) ✅ 24×7 allowed
promotional Marketing/offers to opted-in users ⏰ 9:00 AM – 9:00 PM IST only
service_explicit Service messages to users who have explicitly opted in ⏰ 9:00 AM – 9:00 PM IST only
✅
Recommendation: For OTPs, login codes, and system notifications — always use sms_type=transactional. It works 24×7 and is not subject to time restrictions.

Rate Limits

To ensure platform stability, API requests are rate-limited per account:

LimitValueError Code
Requests per minute 60 requests / minute (configurable by admin) 1009 — Rate limit exceeded
💡
When you hit the rate limit, wait 60 seconds and retry. Implement exponential backoff in your application for robust handling.
Retry Logic (Recommended)
PHP
function sendSmsWithRetry($params, $maxRetries = 3): array {
    $attempt = 0;
    while ($attempt < $maxRetries) {
        $response = sendSms($params);
        if ($response['success'] || http_response_code() !== 429) {
            return $response;
        }
        $delay = (2 ** $attempt) * 1000000; // exponential backoff (microseconds)
        usleep($delay);
        $attempt++;
    }
    return ['success' => false, 'error' => 'Max retries exceeded'];
}

Multi-Template Integration Guide

If your application uses multiple message templates (e.g., License Management, ERP, CRM), each template must have its own DLT-registered template_id. Always pass the matching template_id for each message type.

📋
Key Rule: Each unique message format (OTP, License Key, Payment Confirmation, etc.) needs a separate DLT-approved template with a unique template_id.

/**
 * Send an SMS using a named template.
 *
 * @param string $templateKey  Key from SMS_TEMPLATES (e.g., 'license_generated')
 * @param string $mobile       10-digit mobile number
 * @param array  $vars         Placeholder replacements (['license_key' => 'XXXX', ...])
 */
function sendTemplateSms($templateKey, $mobile, array $vars): array {
    $config = SMS_TEMPLATES[$templateKey] ?? null;

    if (!$config) {
        throw new \InvalidArgumentException("Unknown SMS template: {$templateKey}");
    }

    // Replace {placeholder} with actual values
    $message = $config['message'];
    foreach ($vars as $key => $value) {
        $message = str_replace("{{$key}}", $value, $message);
    }

    $postData = http_build_query([
        'api_key'     => 'sk_your_api_key',
        'mobile'      => $mobile,
        'message'     => $message,
        'sender_id'   => 'MYSEND',
        'entity_id'   => 'YOUR_ENTITY_ID',
        'template_id' => $config['template_id'],
        'sms_type'    => 'transactional',
    ]);

    $ch = curl_init('{BASE_URL}/api/send.php');
    curl_setopt_array($ch, [
        CURLOPT_POST           => true,
        CURLOPT_POSTFIELDS     => $postData,
        CURLOPT_RETURNTRANSFER => true,
        CURLOPT_TIMEOUT        => 30,
    ]);
    $response = json_decode(curl_exec($ch), true);
    curl_close($ch);
    return $response;
}

Java — HttpURLConnection
import java.net.*;
import java.io.*;
import org.json.*;

public class SmsGateway {

    private static final String BASE_URL = "{BASE_URL}";
    private static final String API_KEY  = "sk_your_api_key";

    public static JSONObject sendSms(String mobile, String message,
                                       String templateId) throws Exception {
        String body = "api_key=" + API_KEY +
                      "&mobile="      + URLEncoder.encode(mobile, "UTF-8") +
                      "&message="     + URLEncoder.encode(message, "UTF-8") +
                      "&sender_id=MYSEND" +
                      "&entity_id=YOUR_ENTITY_ID" +
                      "&template_id=" + templateId +
                      "&sms_type=transactional";

        URL url = new URL(BASE_URL + "/api/send.php");
        HttpURLConnection conn = (HttpURLConnection) url.openConnection();
        conn.setRequestMethod("POST");
        conn.setDoOutput(true);
        conn.setConnectTimeout(10000);
        conn.setReadTimeout(30000);

        try (OutputStream os = conn.getOutputStream()) {
            os.write(body.getBytes("UTF-8"));
        }

        BufferedReader br = new BufferedReader(
            new InputStreamReader(conn.getInputStream(), "UTF-8"));
        StringBuilder sb = new StringBuilder();
        String line;
        while ((line = br.readLine()) != null) sb.append(line);

        return new JSONObject(sb.toString());
    }

    public static void main(String[] args) throws Exception {
        JSONObject result = sendSms("9876543210",
            "Your OTP is 123456. Valid for 5 minutes.", "1007XXXXXXXXX001");

        if (result.getBoolean("success")) {
            System.out.println("Sent! Balance: ₹" + result.getDouble("balance"));
        } else {
            System.out.println("Error: " + result.getString("error"));
        }
    }
}
C# — HttpClient
using System.Net.Http;
using System.Text.Json;

public class SmsGateway {
    private const string BASE_URL = "{BASE_URL}";
    private const string API_KEY  = "sk_your_api_key";

    private static readonly HttpClient _client = new() { Timeout = TimeSpan.FromSeconds(30) };

    public static async Task<JsonElement> SendSmsAsync(
        string mobile, string message, string templateId, string smsType = "transactional") {

        var form = new FormUrlEncodedContent(new[] {
            new KeyValuePair<string,string>("api_key",     API_KEY),
            new KeyValuePair<string,string>("mobile",      mobile),
            new KeyValuePair<string,string>("message",     message),
            new KeyValuePair<string,string>("sender_id",   "MYSEND"),
            new KeyValuePair<string,string>("entity_id",   "YOUR_ENTITY_ID"),
            new KeyValuePair<string,string>("template_id", templateId),
            new KeyValuePair<string,string>("sms_type",    smsType),
        });

        var response = await _client.PostAsync($"{BASE_URL}/api/send.php", form);
        var content  = await response.Content.ReadAsStringAsync();
        return JsonSerializer.Deserialize<JsonElement>(content);
    }

    // Usage
    public static async Task Main() {
        var result = await SendSmsAsync(
            "9876543210",
            "Your OTP is 123456. Valid for 5 minutes.",
            "1007XXXXXXXXX001"
        );
        if (result.GetProperty("success").GetBoolean())
            Console.WriteLine($"Sent! Balance: ₹{result.GetProperty("balance").GetDouble()}");
        else
            Console.WriteLine($"Error: {result.GetProperty("error").GetString()}");
    }
}

Best Practices

Security
  • Never hardcode your API Key in source code. Use environment variables or a config file outside the web root.
  • Always make API calls from your server — never from browser-side JavaScript.
  • Rotate your API Key periodically from the dashboard.
  • Log all SMS API calls (mobile, template, status) for audit purposes.
Performance
  • For bulk sends (e.g., 1000+ users), add a 100–200ms delay between requests to stay within rate limits.
  • Check balance before sending large batches. Use the balance API proactively.
  • Implement async queuing (e.g., Redis queue, Laravel Queues) for high-volume notifications.
Reliability
  • Always check success: true in the JSON response before assuming delivery.
  • Log failed SMS attempts and implement retry logic for transient errors.
  • Set a reasonable timeout (30 seconds) on all API calls.

DLT Compliance (India)

📜
TRAI Mandate: All commercial SMS sent in India must comply with the DLT (Distributed Ledger Technology) framework as mandated by TRAI. Non-compliant messages will be blocked by telecom operators.
DLT Registration Checklist
ItemDescriptionWhere to Get
Entity ID Unique ID assigned to your business on DLT DLT portal (Jio, Airtel, Vi, BSNL)
Sender ID 6-character ID shown as sender name Register on DLT portal under Headers
Template ID Unique ID for each approved message template Register each template on DLT portal
❌
Common Mistake: Do not send a message that differs from the DLT-approved template content. Telecom operators will block mismatched messages.
